US-001Submit today's reportpending

Entry point

Home route /. The report form is the top surface; today has no report yet.

User action sequence

  1. Member opens / and sees an empty report form, date prefilled to today.
  2. Member types the body, optional blockers, and picks a mood.
  3. Member presses Submit report.

Expected visible feedback

  • Loading: the Submit button shows a saving state and is disabled.
  • Success: the form collapses to a "Today's report" summary; a new card appears at the top of the feed.
  • Failure: an inline error keeps the typed text.
Edge: a body over 1000 characters is blocked before submit.
Prototype implication: show the empty form, the saving state, and the post-submit summary.
US-002Edit today's own reportpending

Entry point

Home route / after today's report exists — a "Today's report" summary with an Edit action.

User action sequence

  1. Member presses Edit on today's report summary.
  2. The form reopens prefilled with the current values.
  3. Member changes the body or blockers and presses Save changes.

Expected visible feedback

  • Success: the matching feed card updates in place; no duplicate card.
  • Failure: inline error; the edited text is preserved.
Edge: Edit is absent on other members' cards and on past-day cards.
Prototype implication: show the summary state and the reopened, prefilled form.
US-003Browse the 14-day team feedpending

Entry point

Home route /, feed section below the form. Same view for members and managers.

User action sequence

  1. User scrolls to the feed under the form.
  2. User reads team report cards, newest first.

Expected visible feedback

  • Loading: skeleton cards while the feed loads.
  • Success: cards with member name, date, body, blockers badge.
  • Empty: a friendly empty state when no reports exist.
Edge: reports older than 14 days are not shown.
Prototype implication: show the populated feed, the loading skeleton, and the empty state.
US-004Filter the feed by memberpending

Entry point

Member selector at the top of the feed section.

User action sequence

  1. User opens the member selector and picks a member.
  2. The feed narrows to that member's reports.
  3. User clears the filter to restore the full team feed.

Expected visible feedback

  • Success: only the selected member's cards remain.
  • Empty: a "no reports for this member" empty state.
Edge: clearing the filter must fully restore the 14-day feed.
Prototype implication: show the feed filtered to one member and the per-filter empty state.
