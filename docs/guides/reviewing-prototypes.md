# Reviewing Prototypes

<sub>[← Keelson Docs](../README.md) · [English](../README.md#english) | [日本語](../README.md#日本語)</sub>

Use `/keel-proto` after `requirements.md`, `user-stories.md`, `us-mock.html`, and `design.md` are approved.

Review `prototype.html` for two things:

- Does it provide the experience described by the user stories and US mock?
- Does it follow the implementation direction and existing UI evidence in `design.md`?

If the experience is wrong, return to `/keel-us`. If the design direction is wrong, return to `/keel-design`.
