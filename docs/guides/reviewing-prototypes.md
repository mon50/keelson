# Reviewing Prototypes

<sub>[← Keelson Docs](../README.md) · English | [日本語](ja/reviewing-prototypes.md)</sub>

Use `/keel-proto` after `01-requirements/requirements.md`, `02-user-stories/user-stories.md`, `02-user-stories/us-mock.html`, and `03-design/design.md` are approved.

Review `04-prototype/prototype.html` for two things:

- Does it provide the experience described by the user stories and US mock?
- Does it follow the implementation direction and existing UI evidence in `03-design/design.md`?

If the experience is wrong, return to `/keel-us`. If the design direction is wrong, return to `/keel-design`.
