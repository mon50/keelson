# 実装検証コントラクト

<sub>[← Keelson ドキュメント](../../README.md) · [English](../verify-contract.md) | 日本語</sub>

**対象読者:** 検証の境界を理解したいユーザー。
**前提条件:** なし。
**到達点:** `/keel-impl` が各タスクで何を検証しなければならないかが分かる。

- **保証すること:** 利用可能なときにタスクローカルのチェックを実行し、変更したファイルを報告し、`05-plan/plan.md` にタスク状態と実装メモを記録する。
- **報告すること:** 由来アーティファクトへのトレーサビリティ、ファイル境界の遵守、テストコマンド、失敗またはスキップしたチェック、残っている手動検証。
- **保証しないこと:** ビジネス上の正しさ、UX 品質、セキュリティ、および人によるレビューが必要な受け入れ基準。
